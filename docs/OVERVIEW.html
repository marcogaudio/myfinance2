myfinance2

Quantitative finance platform for automated daily analysis of Borsa Italiana (Italian stock market) equities, running end-to-end via GitHub Actions.

1. Project Summary

PropertyValue
MarketBorsa Italiana (Milan, .MI tickers)
BenchmarkFTSEMIB.MI — FTSE MIB Index
Data sourceYahoo Finance via yfinance
Data formatParquet — long/tidy format
Signal typesBreakout · Turtle Trader · Triple MA Crossover · Floor/Ceiling
AutomationGitHub Actions — Mon–Fri after market close (21:00 UTC)
LanguagePython 3.11
Core idea: every day after Borsa Italiana closes, the pipeline automatically downloads new price data, computes trading signals on relative prices (stock vs FTSE MIB), calculates ATR-based stop-losses, and produces an actionable trading dashboard committed back to the repository.

4. Scripts Reference

get_daily_ohlc_data.py

Downloads the current day's OHLC bar for all tickers in ticker.xlsx.

  • Uses algoshort.YFinanceDataHandler with chunked multi-threaded downloads (chunk_size=20, threads=True)
  • use_cache=False — always fetches fresh data
  • Output: data/ohlc/today/it/ohlc_data.parquet

get_historical_ohlc_data.py

One-time (or periodic reset) bulk download from 2016-01-01 to present. After first pass, detects symbols with zero rows and retries individually.

analyze_stock.py + pipeline.py

Main analysis pipeline. analyze_stock.py is the entry point; pipeline.py contains all pure transformation functions.

# Execution order (analyze_stock.py)
cfg     = load_config("config.json")
ohlc, symbols = load_data(historical_parquet, benchmark)
dfs     = build_symbol_dfs(ohlc, symbols)
dfs     = compute_relative_prices(dfs, benchmark_df)   # normalize vs FTSEMIB.MI
dfs, signals = generate_all_signals(dfs, ...)          # Breakout, Turtle, MA, FC
dfs     = calculate_returns(dfs, signals)
dfs     = calculate_stop_losses(dfs, signals, atr_window=14, atr_multiplier=2.0)
save_results(dfs, output_path)
Design principle: each function in pipeline.py is a stateless DataFrame → DataFrame transformation — independently testable and reorderable. No mutable global state.

trading_report.py

Reads analysis_results.parquet and produces the trading dashboard. Detects signal columns automatically: any column X where X_stop_loss also exists. Outputs two sections:

  1. Actionable Trades — Enter/Exit events in the last 4 trading days
  2. Latest Status — per-signal detail for active signals on the latest bar

generate_symbol_notebooks.py

Generates one Jupyter notebook per symbol using papermill. Template: notebooks/symbol_analysis_template.ipynb. Injects SYMBOL parameter per execution. Graceful failure: errors per symbol are logged and skipped.

5. Signal Generation

5.1 Relative Price Normalization

All signals run on relative prices, not raw prices. This removes the systematic benchmark component and isolates alpha.

bmfx   = benchmark_close / benchmark_close[0]   # rebase to 1.0 at first date

ropen  = open  / bmfx
rhigh  = high  / bmfx
rlow   = low   / bmfx
rclose = close / bmfx
Example: if a stock doubled while the benchmark stayed flat, rclose doubles. If both moved identically, rclose stays at 1.0. Signals on relative prices fire only when a stock is outperforming or underperforming the index.

5.2 Breakout Signal

rbo_150  Breakout

A new N-bar high on relative prices triggers a long; a new N-bar low triggers a short. Signal persists (forward-filled) until the opposite extreme is breached.

rhi_N = rolling_max(rhigh, N)
rlo_N = rolling_min(rlow,  N)

if   rhigh == rhi_N  →  signal = +1   (new N-bar high → long)
elif rlow  == rlo_N  →  signal = -1   (new N-bar low  → short)
else                →  forward-fill previous signal

Active parameter: bo_window = 150  |  Output: rhi_150, rlo_150, rbo_150

5.3 Turtle Trader Signal

rtt_5020  Turtle

Dual-window breakout: the slow window (50) sets the direction; the fast window (20) acts as a trailing stop. When the fast breakout reverses, the position closes even if the slow signal hasn't flipped — limiting drawdown.

slow_regime = breakout(rhigh, rlow, window=50)   # direction
fast_regime = breakout(rhigh, rlow, window=20)   # trailing stop

rtt_5020:
  → +1 if slow=+1 AND fast=+1   (long, trailing stop not hit)
  → -1 if slow=-1 AND fast=-1   (short, trailing stop not hit)
  →  0 otherwise                  (closed by trailing stop)

Parameters: fast=20, slow=50  |  Outputs: rbo_20, rbo_50, rtt_5020

5.4 Triple MA Crossover

rsma_50100150  /  rema_50100150  MA Cross

Three moving averages (50, 100, 150) must all align for a signal. The final value is the product of two pairwise crossovers — requiring full stack alignment.

sm = sign(MA_50  − MA_100)   # short vs medium: +1 or -1
ml = sign(MA_100 − MA_150)   # medium vs long:  +1 or -1

final = sm × ml
  → +1  if sm=+1 AND ml=+1  (50 > 100 > 150 — fully bullish)
  → -1  if sm=-1 AND ml=-1  (50 < 100 < 150 — fully bearish)
  →  0  if mixed             (conflicting crossovers)

Run for both sma and ema, producing 6 columns total.

5.5 Floor/Ceiling Regime (rrg)

Identifies support/resistance regimes using swing-high/swing-low detection with volatility bands. Computed by algoshort.RegimeFC. Parameters: vlty_n=63, dist_pct=0.05, retrace_pct=0.05, threshold=1.5.

5.7 Columns added per signal after full pipeline

<signal>              →  ±1 / 0  (raw signal value)
<signal>_chg1D        →  daily price change × lagged signal
<signal>_returns      →  pct_change × lagged signal
<signal>_log_returns  →  log(1 + pct_change) × lagged signal
<signal>_cumul        →  exp(cumsum(log_returns)) − 1
<signal>_stop_loss    →  ATR-based stop-loss level
Lagged signal (no look-ahead bias): returns use signal.shift(1) — today's signal is acted on at tomorrow's open, not today's close.

6. Stop-Loss and Risk

ATR Stop-Loss

# True Range
TR = max(
    high − low,
    |high − close_prev|,
    |low  − close_prev|
)

# Average True Range
ATR(14) = rolling_mean(TR, window=14)

# Stop-Loss levels
Long  position:  stop_loss = close − (ATR × 2.0)
Short position:  stop_loss = close + (ATR × 2.0)
Flat:            stop_loss = NaN
NaN warm-up handling: ATR requires 14 bars before producing a valid value. Early NaN rows are patched with ffill().bfill() so position sizing never receives NaN inputs.

Risk Percent (risk_pct)

risk_pct = |price − stop_loss| / price × 100
  • Only meaningful on Enter signals (sizing into a new position)
  • NaN on Exit signals (position closed, no forward risk)
  • Represents: "if the stop-loss is hit immediately, what % of position value is lost"

Verification (2026-03-06)

CED.MI: price=1.860, stop=1.7957 → |1.860 − 1.7957| / 1.860 = 3.46% ✓
ENV.MI: price=4.140, stop=3.8571 → |4.140 − 3.8571| / 4.140 = 6.83% ✓

Action taxonomy

ActionMeaning

report.txt vs dashboard.xlsx discrepancy: The text report shows EXIT SHORT + ENTER LONG as two events (from rtt_5020), while the dashboard shows FLIP: SHORT → LONG for the same move under rbo_20/rbo_50. The Turtle signal passes through 0 (flat) between positions; Breakout transitions directly.
